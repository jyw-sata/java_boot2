# java_boot2

간단한 멀티 에이전트 파이프라인 예제입니다.
ChatGPT, Claude, Gemini, Cursor AI를 단계별로 호출하여
요구사항 설계 → 모듈별 개발 → 빌드·테스트 순서로 진행하는 구조를 시뮬레이션합니다.

## 파이프라인 스크립트 실행
```
python multi_agent_pipeline.py
```
실행 시 각 단계의 에이전트가 입력을 받아 출력으로 넘기는 과정을
터미널에서 확인할 수 있습니다.

## API 키 관리
- `.env.example`을 복사해 `.env` 파일을 만든 뒤 각 모델의 키를 채워 넣습니다.
- `python-dotenv`가 `.env`를 읽어 환경 변수로 로드하며, 스크립트에서 자동 사용됩니다.
- 버전 관리에 `.env`가 포함되지 않도록 `.gitignore`에 등록되어 있습니다.

## 입력 및 파일 구조
- `multi_agent_pipeline.py` : 파이프라인 본문
- 필요 시 `spec` 문자열을 수정하여 다른 요구사항으로 테스트할 수 있습니다.
- 실제 환경에서는 각 에이전트를 API 키와 함께 호출하고, 결과는 Git 리포지토리에
  모듈별 디렉터리(`docs/`, `src/`, `tests/`)로 분리 관리하는 방식을 권장합니다.

## 빌드 및 테스트
- 현재 스크립트는 시뮬레이션이므로 별도 빌드 도구 없이 실행합니다.
- 실제 프로젝트에서는 CI에서 `./gradlew build` 등의 명령을 실행하고,
  실패 로그를 다시 에이전트에게 전달하여 수정안을 받는 흐름을 구성할 수 있습니다.

## 추가 팁
- 설계 문서와 코드, 테스트를 각각 다른 에이전트에게 맡기면 독립적으로 작업할 수 있습니다.
- CI 로그나 빌드 결과를 요약하여 다시 LLM에게 보내면 자동 수정 루프를 구성할 수 있습니다.
